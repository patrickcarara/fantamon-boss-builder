PRIMEIRO ACESSO / CRIAR ADMIN
-----------------------------
1. Publique os arquivos no GitHub.
2. Abra o site.
3. Clique em "Copiar ID deste dispositivo".
4. No Firestore, crie manualmente:
   coleção: users
   documento: COLE_AQUI_O_UID
   campo: role
   tipo: string
   valor: admin
5. Atualize o site com Ctrl+F5.
6. O status deve mudar para "Administrador".

REGRAS
------
Substitua as regras atuais do Firestore pelo conteúdo de firestore.rules.txt
e clique em Publicar.

CONVITES
--------
Como admin, clique em "Gerar convite único".
O link gerado contém ?convite=CODIGO.
Envie para uma pessoa.
Ao abrir o link, aquele dispositivo vira editor e o convite é consumido.
O mesmo convite não pode ser usado novamente.

VISITANTES
----------
Qualquer visitante pode visualizar Bosses e estratégias.
Somente admin/editor pode alterar.

OBSERVAÇÃO
----------
A autenticação é anônima: não existe tela de login ou senha.
Se um editor limpar os dados do navegador ou trocar de dispositivo,
precisará receber um novo convite.
